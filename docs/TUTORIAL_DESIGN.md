# Stock Market — Дизайн системы обучения (Onboarding / Tutorial)

## Концепция

**Модальное окно «Руководство игрока»** — сайдбар-навигация слева (H1-заголовки механик) + детальная панель справа. Открывается автоматически при первом заходе в игру, доступно позже через иконку «?» в хедере дашборда.

---

## Структура навигации (Left Sidebar — H1)

| № | Заголовок (H1) | Ключ | Описание раздела |
|---|----------------|------|------------------|
| 1 | **🎯 Цель игры и Мечта** | `dream` | Что такое мечта, этапы, как выиграть |
| 2 | **📅 Ход игры и Экономика** | `turn-economy` | Пошаговая структура, зарплата, расходы, пассивный доход |
| 3 | **📈 Биржа и Акции** | `stocks` | Покупка/продажа, грейды, комиссии, дивиденды, IPO |
| 4 | **🏠 Недвижимость и Переговоры** | `real-estate` | Офферы BUY/SELL, рассрочка, торги D20, досрочное погашение |
| 5 | **🤝 Сделки с NPC (Deals)** | `deals` | Bundle-to-bundle обмены, репутация, выгода % |
| 6 | **🎲 OTC-сделки** | `otc` | Внебиржевые предложения ботов |
| 7 | **🧠 Навыки и Прогрессия** | `skills` | 4 навыка, апгрейды, слоты недвижимости |
| 8 | **📰 Новости и Инсайдеры** | `news` | Виды новостей, влияние на рынок, инсайдерские советы |

### 2. 📅 Ход игры и Экономика (`turn-economy`)

```
┌─────────────────────────────────────────────────────────────┐
│  Игра пошаговая. Кнопка «Завершить ход» (справа вверху)    │
│  запускает расчёт следующего хода.                          │
├─────────────────────────────────────────────────────────────┤
│  🔄 ЧТО ПРОИСХОДИТ НА ХОДЕ (Pipeline)                       │
│  1. advance-step   — step++, проверка банкротства           │
│  2. economy        — ДЕНЕЖНЫЙ ПОТОК (см. ниже)              │
│  3. market-turn    — Биржа: цены, дивиденды, сентимент      │
│  4. turn-content   — Контент хода (цикл 3 хода):            │
│     • Ход 1: Новости (Stock/Rumor/Insider)                  │
│     • Ход 2: Сделка с NPC (Deal)                            │
│     • Ход 3: Недвижимость (Property Offer)                  │
│  5. property-expiry — Удаление просроченных офферов         │
│                                                             │
│  💰 ЭКОНОМИКА ХОДА (PassiveIncomeService)                   │
│  ┌──────────────────┬────────────────────────────────────┐  │
│  │ Зарплата         │ Каждый 3-й ход (isSalaryTurn)      │  │
│  │                  │ = salary × (1 + 0.1×(profLvl-1))   │  │
│  │                  │ +10%/ур. квалификации (навык)      │  │
│  ├──────────────────┼────────────────────────────────────┤  │
│  │ Расходы на жизнь │ Случайные чеки каждый ход          │  │
│  │                  │ (еда, ремонт, лечение, развлечения)│  │
│  ├──────────────────┼────────────────────────────────────┤  │
│  │ Рассрочки        │ Месячные платежи по ипотеке/кредитам│  │
│  │                  │ (только если !isPaidOff)           │  │
│  ├──────────────────┼────────────────────────────────────┤  │
│  │ Пассивный доход  │ Парсится из special:               │  │
│  │                  │ "пассивный доход 500/ход"          │  │
│  │                  │ Недвижимость: склад, караван, яхта │  │
│  ├──────────────────┼────────────────────────────────────┤  │
│  │ ИТОГО (netChange)│ salary - living - installments     │  │
│  │                  │ + passiveIncome                    │  │
│  └──────────────────┴────────────────────────────────────┘  │
│                                                             │
│  ⚠️ БАНКРОТСТВО: если balance + netChange < 0 →             │
│  game.status = COMPLETED (проигрыш)                         │
│                                                             │
│  📊 ПРОГНОЗ СЛЕДУЮЩЕГО ХОДА — правая колонка дашборда,      │
│  блок «Next Turn Forecast» (зелёные/красные строки)        │
└─────────────────────────────────────────────────────────────┘
```

### 3. 📈 Биржа и Акции (`stocks`)

```
┌─────────────────────────────────────────────────────────────┐
│  Вкладка «Биржа» (центр дашборда, таб Exchange)             │
├─────────────────────────────────────────────────────────────┤
│  🏷️ ГРЕЙДЫ АКЦИЙ (F → E → D → C → B → A)                    │
│  ┌──────┬──────────┬──────────┬──────────┬────────────────┐ │
│  │ Грейд│ Мин.Банк │ Мин.Реп  │ На бирже │ Ценовой диапаз │ │
│  ├──────┼──────────┼──────────┼──────────┼────────────────┤ │
│  │ F    │ 1        │ 1        │ Да       │ Низкий         │ │
│  │ E    │ 1        │ 2        │ Да       │ Низкий+        │ │
│  │ D    │ 2        │ 3        │ Да       │ Средний        │ │
│  │ C    │ 3        │ 4        │ Да       │ Средний+       │ │
│  │ B    │ 4        │ 6        │ Да       │ Высокий        │ │
│  │ A    │ 5        │ 8        │ Да       │ Премиум        │ │
│  └──────┴──────────┴──────────┴──────────┴────────────────┘ │
│  🔒 ЗАМОК на карточке = недоступно (банк/реп/не на бирже)   │
│                                                             │
│  📊 ЦЕНООБРАЗОВАНИЕ (каждый ход)                             │
│  newPrice = currentPrice × (1 + baseVolatility             │
│    + newsPressure + sectorMomentum + marketSentiment)      │
│  • Новости создают давление (NewsPressure) на тикеры/секторы│
│  • Сентимент: глобальный [-1,1], обновляется каждый ход     │
│  • Секторный моментум: по 6 секторам, затухает каждый ход   │
│                                                             │
│  💵 ПОКУПКА / ПРОДАЖА                                        │
│  • Кнопка на карточке акции → модалка кол-ва                │
│  • Покупка: списывается balance, создаётся/обновляется Stock│
│  • Продажа: комиссия = 10% - tradingLevel×1% (мин 5%)      │
│    Навык Trading 6 лвл → 5% комиссия                        │
│  • PnL считается от purchasePrice (взвешенный средний)     │
│                                                             │
│  🎁 ДИВИДЕНДЫ                                                │
│  • Профиль роллится при листинге: yield%, frequency, growth│
│  • Выплата автоматически на баланс, новость STOCK_DIVIDEND │
│  • В портфолио: turnsUntilDividend, dividendPerShare       │
│                                                             │
│  🚀 IPO                                                      │
│  • Анонс → Окно подписки (мин/макс) → Размещение → Биржа   │
│  • Таб IPO на бирже: прогресс подписки, annoncéAt/ipoAt    │
│                                                             │
│  📈 ГРАФИКИ (Sparkline) — 20 последних ходов, прогрев      │
│  истории при старте игры                                    │
└─────────────────────────────────────────────────────────────┘
```

### 4. 🏠 Недвижимость и Переговоры (`real-estate`)

```
┌─────────────────────────────────────────────────────────────┐
│  Вкладка «Недвижимость» (центр, таб Real Estate)            │
│  + карточки в правой колонке (Property Offers)              │
├─────────────────────────────────────────────────────────────┤
│  📋 ТИПЫ ОФФЕРОВ                                             │
│  ┌──────┬────────────────────────────────────────────────┐  │
│  │ SELL │ Бот ПОКУПАЕТ у вас (нужен предмет в инвентаре, │  │
│  │      │ без активной рассрочки). Вы получаете кэш.      │  │
│  ├──────┼────────────────────────────────────────────────┤  │
│  │ BUY  │ Бот ПРОДАЁТ вам (новый предмет в инвентарь).   │  │
│  │      │ Оплата: полная или в рассрочку.                 │  │
│  └──────┴────────────────────────────────────────────────┘  │
│                                                             │
│  🎨 ГРЕЙДЫ ПРИБЫЛЬНОСТИ (F → A) — цветной бейдж на карточке│
│  ┌──────┬──────────────┬──────────────┬─────────────────┐  │
│  │ Грейд│ Профит %     │ Взнос %      │ Мин. Банк лвл   │  │
│  ├──────┼──────────────┼──────────────┼─────────────────┤  │
│  │ F    │ -20%..-5%    │ 50%          │ 1               │  │
│  │ E    │ -5%..+5%     │ 40%          │ 1               │  │
│  │ D    │ +5%..+15%    │ 30%          │ 2               │  │
│  │ C    │ +15%..+25%   │ 20%          │ 3               │  │
│  │ B    │ +25%..+40%   │ 15%          │ 4               │  │
│  │ A    │ +40%..+60%   │ 10%          │ 5               │  │
│  └──────┴──────────────┴──────────────┴─────────────────┘  │
│  🔥 isHot — редкие выгодные офферы (пометка огоньком)       │
│                                                             │
│  💳 РАССРОЧКА (Installment Purchase)                        │
│  • План: calcInstallmentPurchasePlan({price, down%, total, │
│    bankingLevel}) — bankingLevel снижает месячный платёж   │
│  • В инвентаре: isInstallment=true, monthlyPayment,        │
│    installmentsTotal, installmentsPaid, isPaidOff          │
│  • Ежемесячно списывается monthlyPayment (фаза economy)    │
│                                                             │
│  🎲 ПЕРЕГОВОРЫ (Negotiate) — кнопка на карточке             │
│  1. Введите % корректировки цены:                           │
│     SELL: -50%..+10%  |  BUY: -10%..+50%                   │
│  2. TradingLevel улучшает условия (normalizeNegotiate%)    │
│  3. Бросок D20: roll = d20 + floor(reputation)             │
│     Target зависит от типа и % корректировки               │
│  4. Успех → pendingNegotiatedPrice сохраняется → кнопка    │
│     «Принять переговоры»                                    │
│  5. Провал → -0.1 репутации, streak сбрасывается           │
│                                                             │
│  📤 ПРОДАЖА С РАССРОЧКОЙ (SELL offer, owned item)           │
│  • calcInstallmentSaleBreakdown(owned, salePrice):         │
│    остаток долга вычитается из цены → чистая сумма на баланс│
│  • Если долг > цены → доплата с баланса (может быть минус)  │
│                                                             │
│  ⚡ ДОСРОЧНОЕ ПОГАШЕНИЕ                                      │
│  • Банк (правая колонка) → кнопка «Погасить %»             │
│  • % от остатка долга, лимит = баланс                      │
│  • applyEarlyInstallmentPayment → увеличивает paid,        │
│    может закрыть isPaidOff = true                          │
│                                                             │
│  🔢 СЛОТЫ НЕДВИЖИМОСТИ                                       │
│  • propertySlotLevel (навык «Слоты имущества»):            │
│    1→1 слот, 2→2, 3→3, 4→4 активных предмета              │
│  • Превышение → нельзя купить новый BUY оффер               │
└─────────────────────────────────────────────────────────────┘
```

### 5. 🤝 Сделки с NPC (Deals) (`deals`)

```
┌─────────────────────────────────────────────────────────────┐
│  Таб «Сделки» (центр) + карточки в правой колонке           │
│  Появляются каждый 3-й ход (фаза turn-content)              │
├─────────────────────────────────────────────────────────────┤
│  🎁 СУТЬ: Bundle-to-bundle обмен с случайным NPC-ботом      │
│  Бот даёт набор ассетов → Вы даёте свой набор               │
│                                                             │
│  📦 ТИПЫ ШАБЛОНОВ (веса генерации)                          │
│  ┌─────────────────────┬─────┬───────────────────────────┐ │
│  │ Шаблон              │ Вес │ Что в бандле              │ │
│  ├─────────────────────┼─────┼───────────────────────────┤ │
│  │ CASH_FOR_STOCK      │ 3   │ Бот: кэш → Вы: 5-30 акций │ │
│  │ PROPERTY_FOR_CASH   │ 2   │ Бот: недвижка → Вы: кэш   │ │
│  │ MIXED_BUNDLE        │ 2   │ Бот: акции(3-15)+кэш      │ │
│  │                     │     │ (2000-15000)+опц.недвижка │ │
│  │ STOCK_PACKAGE       │ 1   │ Пакет акций за кэш        │ │
│  │ DREAM_HELPER        │ 1   │ Бот даёт предмет мечты    │ │
│  │ LIQUIDITY           │ 1   │ Быстрый обмен ликвидности │ │
│  └─────────────────────┴─────┴───────────────────────────┘ │
│                                                             │
│  ⚙️ ПАРАМЕТРЫ СДЕЛКИ                                        │
│  • requiredReputation = clamp(player.rep × 0.7, 1, 10)    │
│  • requiredTradingLevel — зависит от грейдов акций в сделке │
│  • expiresInTurns = 2-4 (счётчик на карточке)              │
│  • playerBenefitPercent — % выгоды (зелёный/красный бейдж) │
│  • reputationPenalty — штраф реп. при плохой сделке        │
│                                                             │
│  ✅ ПРИНЯТИЕ (DealService.accept)                           │
│  1. Проверки: ACTIVE, не просрочена, реп ≥ required,       │
│     tradingLevel ≥ required                                │
│  2. Валидация ваших ассетов:                                │
│     - CASH: balance ≥ сумма                                 │
│     - STOCK: тикер и кол-во в портфеле                      │
│     - PROPERTY: в инвентаре, !isInstallment || isPaidOff   │
│  3. Транзакция: списание → зачисление → репутация +статы   │
│  4. Статус → ACCEPTED, новость DEAL_OFFER                   │
│                                                             │
│  ❌ ОТКЛОНЕНИЕ — статус REJECTED, сделка исчезает           │
│                                                             │
│  💡 СОВЕТ: Оценивайте playerBenefitPercent.                │
│  Зелёный > 0% = выгода. Красный < 0% = убыток,             │
│  но может быть полезно для мечты (нужный актив).           │
└─────────────────────────────────────────────────────────────┘
```

/// • Цена может отличаться от биржевой (премия/скидка)      │

/// • После принятия исчезает из списка                      │
└─────────────────────────────────────────────────────────────┘
```

### 7. 🧠 Навыки и Прогрессия (`skills`)

```
┌─────────────────────────────────────────────────────────────┐
│  Левая колонка дашборда → секция «Навыки»                   │
│  4 независимых дерева, апгрейд за баланс                    │
├─────────────────────────────────────────────────────────────┤
│  📚 СПИСОК НАВЫКОВ                                          │
│  ┌──────────────────┬──────┬────────────────────────────┐  │
│  │ Навык            │ Макс │ Эффект за уровень          │  │
│  ├──────────────────┼──────┼────────────────────────────┤  │
│  │ 🎓 Qualification │ 10   │ +10% к зарплате            │  │
│  │ (Повышение квал.)│      │ +2% шанс инсайдерской нов. │  │
│  ├──────────────────┼──────┼────────────────────────────┤  │
│  │ 🏦 Banking       │ 6    │ -2% ставка по кредитам     │  │
│  │ (Курсы банка)    │      │ Доступ к грейдам недвиж.   │  │
│  │                  │      │ (F→E→D→C→B→A по лвлам)     │  │
│  ├──────────────────┼──────┼────────────────────────────┤  │
│  │ 📊 Trading       │ 6    │ Комиссия продажи: 10%→5%   │  │
│  │ (Курс трейдинга) │      │ Торг недвижимостью до 50%  │  │
│  ├──────────────────┼──────┼────────────────────────────┤  │
│  │ 🏠 Property Slots│ 4    │ +1 слот недвижимости/ур.   │  │
│  │ (Слоты имущества)│      │ (макс 4 активных предмета) │  │
│  └──────────────────┴──────┴────────────────────────────┘  │
│                                                             │
│  💰 ЦЕНА АПГРЕЙДА                                           │
│  basePrice × (level^1.5) + зависимость от salary           │

### 9. 💰 Банк, Кредиты и Репутация (`bank-reputation`)

```
┌─────────────────────────────────────────────────────────────┐
│  Правая колонка → блок «Банк» (активные кредиты, погашенные)│
│  Репутация — в профиле персонажа (левая колонка)            │
├─────────────────────────────────────────────────────────────┤
/// 🏦 БАНК / РАССРОЧКИ                                         │
/// • Активные: itemRef, name, остаток долга, платёж/ход,     │
///   paid/total, прогресс-бар, кнопка «Погасить %»          │
/// • Погашенные: история выкупленных активов                 │
/// • Итоги: totalDebt, paymentPerTurn, turnsUntilNextCharge │
///                                                             │
/// 📊 РЕФИНАНСИРОВАНИЕ (Banking Level)                         │
/// • Каждый уровень Banking = -2% к ставке                  │
/// • Влияет на monthlyPayment при покупке в рассрочку       │
/// • Формула в calcInstallmentPurchasePlan                  │
///                                                             ///
/// ⭐ РЕПУТАЦИЯ (1.0 – 10.0)                                   ///
/// • Старт: 3.0                                               ///
/// • Успешная сделка (benefit ≥ 0): +0.2                     ///
/// • Успешная сделка (benefit < 0): -reputationPenalty      ///
/// • Провал переговоров: -0.1, streak reset                      ///
/// • Успешная торговля недвижимостью: бонус за streak       ///
/// • clamp(1, 10) после каждого изменения                   ///
///                                                             ///
/// 🎯 ВЛИЯНИЕ РЕПУТАЦИИ:                                       ///
/// • Доступ к грейдам акций (minReputation)                  ///
/// • requiredReputation в NPC-сделках (≈ rep × 0.7)         ///
/// • D20 бонус в переговорах: roll = d20 + floor(rep)       ///
/// • Влияет на шанс инсайдера (профессия + репутация)       ///
///                                                             ///
/// 💀 БАНКРОТСТВО                                              ///
/// • Если после хода balance < 0 → game.status = COMPLETED  ///
/// • Предупреждение перед окончанием хода (модалка)         ///
/// • В Next Turn Forecast: красная строка «Банкротство»     ///
└─────────────────────────────────────────────────────────────┘
```

| Момент | Решение |
|--------|---------|
| **Первый вход** | Модалка открывается автоматически после загрузки дашборда, фокус на сайдбаре |

| **Аккордеон** | Только один раздел открыт за раз (radio-group поведение) |

| **Поиск** | Поле поиска над сайдбаром — фильтрует H1 по ключевым словам |
| **Контекстная помощь** | В модалках сделок/недвижимости — маленькая иконка «i» → deep-link в Tutorial (scroll to section) |
| **Мобильная** | Сайдбар — bottom sheet, панель — фуллскрин, свайп для закрытия |
| **Анимации** | Framer Motion: slide-in модалки, accordion collapse/expand, fade контента |

## Контентная стратегия

### Тон голосования
- **Дружелюбный наставник** — «Давайте разберёмся», «Совет:», «Обратите внимание»
- **Конкретика** — цифры, формулы, пороги (не «много», а «>50%»)
- **Визуальные якоря** — эмодзи-маркеры (🎯 💰 📈 ⚠️ 💡) для сканирования

### Локализация
- `tutorialData.ts` — массив объектов `{ key, title, icon, contentMarkdown }`
- Готовность под i18n: вынести строки в `locales/ru/tutorial.json`

### Версионирование
- `localStorage` ключ включает версию (`v1`)
- При мажорном обновлении контента — инкремент версии → повторный онбординг
